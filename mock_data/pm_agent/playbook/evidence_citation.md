# Evidence Citation Playbook

Use this playbook to keep PM Agent answers grounded. Every important claim should trace to a source.

## Citation Rules

Every blocker must cite at least one evidence ID:

- Jira key, such as JIRA-142.
- Pull request ID, such as PR-88.
- Customer account name, such as Acme Commerce.
- Email, Slack, task, calendar, or meeting item when available.
- Product docs or launch criteria when explaining policy.

## Claim Quality

Strong evidence:

- Direct Jira status, priority, blocker field, or launch criteria blocker flag.
- GitHub PR status, failing check, or reviewer state.
- Customer escalation with account, deadline, revenue, and requested action.
- Calendar/task evidence showing missing owner or deadline.

Weak evidence:

- Old comments without current status.
- Unassigned tasks with no due date.
- Informal sentiment without linked blocker.

## Missing Evidence

If evidence is missing, write "Missing evidence" and name the missing artifact:

- Missing QA sign-off.
- Missing owner.
- Missing rollback plan.
- Missing customer comms.
- Missing passing CI.

Do not invent evidence IDs or customer commitments.
