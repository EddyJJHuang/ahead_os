# Customer Escalations Playbook

Customer escalations should be handled with a clear owner, deadline, evidence plan, and fallback. Avoid promising GA dates when the launch recommendation is hold or conditional go.

## Enterprise Escalation Handling

For renewal-critical accounts, provide proof points rather than vague status. Examples include passing load-test results, merged pull requests, support runbook links, signed evidence packets, and rollback plans. If proof is not ready, send an exception plan with the next evidence date.

## Communication Rules

Name the blocker plainly. Explain the mitigation. Give a next update time. Separate what is shipped, what is behind a feature flag, and what is not ready. For red-health accounts, route the update through customer success and the executive sponsor.
